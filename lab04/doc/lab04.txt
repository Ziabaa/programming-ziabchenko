1)Завдання:  Визначити, чи є ціле 6-значне число "щасливим" квитком ("щасливий квиток" – квиток, в якому сума першої 
половини чисел номера дорівнює сумі другої половини.
Основна частина:
- опис роботи основної функції: щоб знайти суму першої половини і другої половинии, потрібно спочатку знайти кожну 
цифру, а потім суму перших трьох порівняти з сумую других трьох.
- перелік вхідних данних:
	number - дане 6-значне число, позитивне ціле число (int);
	result - результат, може бути лише 1 або 0, тому позитивне ціле число (int);
- дослідження результатів роботи програми
	1)first_number - перше число. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні number = 234027, перше число повинно складати: number / 100000;
	2)second_number - друге число. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні number = 234027, друге число повинно складати: (number / 10000) % 10;
	3)third_number - третьє число. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні number = 234027, третьє число повинно складати: (number / 1000) % 10;
	4)fourth_number - четверте число. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні number = 234027, четверте число повинно складати: (number / 100) % 10;
	5)fifth_number - п'яте число. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні number = 234027, п'яте число повинно складати: (number / 10) % 10;
	6)sixth_number - шосте число. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні number = 234027, шосте число повинно складати: number / 100000;
	7)sum_first_tree_number - сума перших 3 чисел. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні first_number = 2, second_number = 3, third_number = 4, сума перших 3 чисел повинна
         складати: first_number + second_number + third_number;
	8)sum_second_tree_number - сума других 3 чисел. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні fourth_number = 0, fifth_number = 2, sixth_number = 7, сума других 3 чисел повинна
         складати: fourth_number + fifth_number + sixth_number; 
	9)використати оператор if, та за тієї умови, якщо сума перших буде дорівннювати сумі других, тоді result = 1, а якщо ні,
         то result = 0.
- дослідження результатів роботи програми 
	1)-sum_first_tree_number - сума перших 3 чисел. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	    Тому, її тип - int - при значенні 2 + 3 + 4 = 9    
          - Для підтвердження коректності роботи програми, зупинено відлагодник на строці з "return 0" та введемо команду
	 "print sum_first_tree_number". Після вводу команди отримали наступне: 
	 (lldb) print sum_first_tree_number
 	 (int) sum_first_tree_number = 9
            Як бачимо, результат співпав, подібність результатів говорить про те, що програма працює коректно.
	2)-sum_second_tree_number - сума других 3 чисел. Так як всі оперуємі числа є цілими, то й результуюча змінна є цілою.
	 Тому, її тип - int - при значенні 0 + 2 + 7 = 9    
          - Для підтвердження коректності роботи програми, зупинено відлагодник на строці з "return 0" та введемо команду
	 "print sum_second_tree_number". Після вводу команди отримали наступне: 
	 (lldb) print sum_second_tree_number
 	 (int) sum_second_tree_number = 9
            Як бачимо, результат співпав, подібність результатів говорить про те, що програма працює коректно.
	9)-Сума перших цифр дорівннює сумі других цифр, тоді result повинен дорівнювати 1
	 - Для підтвердження коректності роботи програми, зупинено відлагодник на строці з "return 0" та введемо команду
	 "print sum_second_tree_number". Після вводу команди отримали наступне: 
	 (lldb) print result
 	 (int) result = 1
	Як бачимо, результат співпав, подібність результатів говорить про те, що програма працює коректно, не маючи ніяких 
	помилок. Також змінивши number на число 234029, через відлагодник ми отримали result = 0, а це вже остаточно підтверджує
	правильність роботи програми.
